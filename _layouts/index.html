What's Included?

Code Base

The Code Base is hosted on Josten's SVN. Look in /examples for specific files. Live versions are referred to in the Catalog. The Code Base consists of:

  1. Markup: Plain HTML. No Templating/processing
  2. Styles: .LESS files compiled into .CSS files
  3. Interaction: JavaScript, jQuery, etc...

Catalog

The Catalog is a list of all the Records in the library.

Docs

The Docs provide a space for:

  • UX rationalle and direction
  • Dev direction as available/necessary

Catalog Records

Library Records are items tracked by the library. They are as follows:

Themes

Themes are collections of styles. Themes can be applied to any type of Record.

Experiences

Experiences are collections of Views.

Views

Views are collections of Components. Contains one-to-many components. Generally known as "pages", but describes what is viewable in a device's viewport.

Components

Components are the meat of the Library. They are reusable parts to make something bigger: a page, a column, etc... Components may contain other Components.

Catalog Attributes

Here's what will help describe Library Records in the Catalog

Type

Can be one of the following:

  • Theme
  • Experience
  • View
  • Component

Name

A record's Name is the key identifying attribute. A record's name should be how it is referenced by in all documentation, discussions, and designs through the business and software development lifecycle.

Experience & Theme

The Experience & Theme that the Record is attached to.

Status

Statuses will define where a Record is in the development lifecycle:

  1. Needs Review: Idea has been created; sketch, Photoshop, whatever...
  2. In Review: Necessary parties review as needed: UX Team, stakeholders, product owners, etc...
  3. Approved: Ready for production
  4. Deprecated: Still supported, but no longer advancing in design
  5. Retired: Not supported
